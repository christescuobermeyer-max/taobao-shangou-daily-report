# 淘宝闪购阶段一批量日报实施计划

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 实现一个 Node.js ESM 脚本，从淘宝闪购服务商 Excel 中提取门店信息，串行抓取昨日接口原始字段，调用大模型生成日报，并先为王清月前两家门店输出测试工作簿。

**Architecture:** `isv-daily-report.mjs` 作为编排入口，内部按 Excel 读取/格式化、CDP 取数、环境变量配置、模型生成和 Excel 写入划分为小函数。取数和本轮模型调用均串行；每家门店的完整原始接口响应写入一个可换行的 Excel 单元格，同时把标准化指标摘要附加到模型输入中。

**Tech Stack:** Node.js 18+ ESM, `ws`, `xlsx` (SheetJS), Node built-in `node:test`, `node:http`, `node:https`, existing `isv-report-lib.mjs`.

## Global Constraints

- 平台按淘宝闪购处理；`open.shop.ele.me`、`ELEME_ISV_COOKIE` 和 `X-Eleme-*` 是实际后端技术标识，保留不改。
- 源文件 `E:\christescuobermeyer\taobaoshangou-shengxiaozhongdianpu\淘宝闪购已建群_按运营分类.xlsx` 只读，不覆盖或改写。
- 本轮命令必须显式使用 `--operator 王清月 --limit 2`，只处理源表第 2、3 行。
- 输出列固定为 `门店ID`、`店铺名称`、`运营`、`昨日数据`、`微信群名`、`昨日日报`。
- 门店 ID 全流程按字符串处理，禁止使用可能产生科学计数法的数值转换。
- `昨日数据` 单元格保存日期、门店 ID、接口店铺名，以及 `data[0]` 全部字段的中文数据名称和原始值。
- 页面搜索必须串行；单店失败记录原因后继续下一家；大模型 API Key、Cookie 不写入日志或输出文件之外的任何位置。
- 取数接口和大模型接口默认失败重试 2 次，间隔 3000ms；重试仍失败才写入失败信息并继续下一家。
- 运行时动态字段 `secretkey`、`uniquekey`、`timestamp`、`bx_et`、`token` 由真实页面生成，不能硬编码或写入 `.env`。
