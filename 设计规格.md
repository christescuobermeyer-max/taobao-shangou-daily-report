# 淘宝闪购阶段一批量日报设计

**日期：** 2026-07-17

## 目标

实现阶段一脚本，读取淘宝闪购已建群工作簿中的门店信息，逐店从已登录的淘宝闪购服务商 Chrome 获取昨日接口原始数据，调用大模型生成运营日报，并将结果写入新的 Excel 文件。

本轮只验证 `王清月` 工作表的前两条数据，不处理其他运营或其他门店。

## 输入

源文件：`E:\christescuobermeyer\taobaoshangou-shengxiaozhongdianpu\淘宝闪购已建群_按运营分类.xlsx`

工作表：`王清月`

源表第 1 行是表头，使用以下字段：

| 输出字段 | 源字段 | 源列 |
|---|---|---|
| 门店ID | 签约门店ID | F |
| 店铺名称 | 签约门店名称 | G |
| 微信群名 | 微信群名 | M |
| 运营 | 运营 | O |

测试范围为源表第 2、3 行：

- `1330475849`：老碰私厨馆(汇川店)
- `1334001820`：蛮掌柜罐罐米线(新桥路店)

门店 ID 必须按字符串处理，避免 Excel 数字格式或科学计数法改变 ID。

## 数据流程

1. 读取源工作簿和指定工作表。
2. 只取测试行对应的两家门店，保留源表中的店铺名称、运营和微信群名。
3. 使用本地时区计算昨日日期，默认格式为 `yyyyMMdd`，并支持命令行覆盖。
4. 连接 Chrome 调试端口，定位淘宝闪购服务商门店分析页面。
5. 在同一个页面中按顺序搜索两个门店 ID，监听 `pageQueryOpenOfflineDetails` 的 POST 响应。
6. 对每个响应保存完整的 `data[0]` 原始对象，不丢弃接口字段。
7. 将每个原始字段转换为对应的中文业务名称，保留原始值；同时补充日期、门店 ID 和接口返回的店铺名，组成发送给大模型的完整数据文本。
8. 通过 `isv-report-lib.mjs` 调用 OpenAI 兼容 API，生成淘宝闪购店铺日报。
9. 写入新的 Excel 文件，不修改源文件。

页面搜索和取数必须串行；本轮两家门店的大模型调用也串行，便于核对每家数据与日报的对应关系。

## Excel 输出

输出文件默认保存到源文件所在目录，文件名为：

`淘宝闪购王清月日报测试.xlsx`

输出表只包含以下六列，顺序固定：

| 列 | 内容 |
|---|---|
| 门店ID | 从源表复制的字符串门店 ID |
| 店铺名称 | 从源表复制的店铺名称 |
| 运营 | 从源表复制的运营姓名 |
| 昨日数据 | 每家店一个单元格，保存完整字段的中文数据名称和原始值 |
| 微信群名 | 从源表复制的微信群名 |
| 昨日日报 | 大模型生成的纯文本日报 |

`昨日数据` 使用单元格内换行，格式为：

```text
昨日日期：20260716
门店ID：1330475849
接口店铺名称：老碰私厨馆(汇川店)
全部昨日数据（中文字段名称）：
营业额：123.45
上周营业额变化值：-10.00
有效订单数：28
```

示例中的字段仅展示格式，实际输出必须写入接口响应行的全部字段，包括接口返回的所有原始值。字段名称使用业务词典和时间、环比、大盘组合规则转换为中文，不再把英文字段名直接交给大模型。

`店铺名称` 保留源表名称；接口返回名称写入 `昨日数据`。如果两者不一致，脚本在控制台给出警告，但不修改源表名称。

## 大模型输入

大模型用户消息包含：

1. 昨日日期、门店 ID、接口店铺名称。
2. 全部字段的中文数据名称和原始值。
3. 由 `extractShopMetrics()` 生成的标准指标摘要，作为关键指标提示。

这样 Excel 中保存的是完整原始数据，同时日报仍能稳定识别营业额、曝光、进店、转化率和订单等核心指标。

API 配置从 `.env` 读取：

- `VECTORENGINE_API_KEY`
- `VECTORENGINE_BASE_URL`
- `VECTORENGINE_MODEL`

服务商登录态使用现有的 `ELEME_ISV_COOKIE`。真实运行时参数 `secretkey`、`uniquekey`、`timestamp`、`bx_et` 和 `token` 必须由页面运行时生成。

## 错误处理

- Chrome 调试端口不可用：立即停止并输出启动提示。
- 找不到淘宝闪购搜索框：提示登录服务商账号后停止。
- 某家门店无匹配数据：该行保留基础信息，在 `昨日数据` 和 `昨日日报` 写入明确错误文本，并继续下一家。
- 接口超时或大模型失败：记录门店 ID 和失败原因，继续下一家。
- 当前测试若任一家失败，输出文件仍然保存，便于检查失败原因。

## 测试验收

自动测试先覆盖纯函数：

- 正确从源表映射 F/G/M/O 四个字段。
- 门店 ID 始终保持字符串。
- 测试范围只返回王清月第 2、3 行。
- 原始响应格式化后包含日期、门店 ID、接口店铺名和全部原始字段。
- 输出列顺序固定为六列。

实际联调验收：

1. 两家门店均成功抓到昨日响应。
2. 每家 `昨日数据` 单元格包含完整原始 JSON。
3. 每家 `昨日日报` 与对应门店匹配，且包含淘宝闪购运营分析内容。
4. 运营和微信群名与源工作表一致。
5. 新 Excel 可正常打开，长文本可换行查看，源 Excel 不发生变化。
