# 淘宝闪购服务商版 · 门店分析接口接入文档

> 把淘宝闪购**服务商控制台**（`open.shop.ele.me`）"代运营 - 数据中心 - 门店分析"的服务商全店概览、门店列表搜索、免登进店、以及进店后单店经营分析昨日概览接入你的软件。
>
> 平台说明：本文面向淘宝闪购；`open.shop.ele.me`、`.ele.me` Cookie 和 `ELEME_ISV_COOKIE` 是淘宝闪购后台当前实际使用的历史技术标识，按抓包结果保留。
> 本文基于服务商单元 `unitId=1132809`、实测门店 `1319751638`（小输胜麺屋）的真实抓包生成，抓包时间 2026-07-17。

**已验证：服务商 cookie 可以登录 `open.shop.ele.me` 控制台，能获取全店门店分析概览、按门店 ID 搜索门店、并通过"免登会话"进入任意名下门店的单店经营分析。**

| 项目 | 值 |
|------|----|
| 控制台 | `https://open.shop.ele.me/manager/base/store-analysis` |
| 数据来源标注 | 页面显示"数据来自生意参谋" |
| 服务商单元 | `unitId = 1132809` |
| 服务商用户 | `openUserId = 100100161380818`，`accountType = member` |

**7 个页面动作 → 真实接口对应关系**（注意不是 1:1）：

| # | 你描述的动作 | 真实触发的接口 | 说明 |
|---|-------------|---------------|------|
| 1 | 进入门店分析页 | `queryOpenRtAnalyzeOverview` + `queryOpenAnalyzeOverview` + `pageQueryOpenOfflineDetails` | 首屏默认加载今日实时+概览+列表 |

## 1. 工作原理

```
你的软件
   │
   │ ① 从本地 .env 取淘宝闪购服务商 cookie（ELEME_ISV_COOKIE）
   │ ② 用 CDP 注入 cookie 到 open.shop.ele.me
   │ ③ 打开门店分析页，页面读取 unitId / openUserId / accountType
   ▼
open.shop.ele.me/manager/base/store-analysis
   │ 页面运行时生成 token、secretkey、uniquekey、timestamp、bx_et
   ▼
lsycm.alibaba.com/api/oShopAnalysis/open/general/*   （服务商全店概览 + 门店列表）
lsycm.alibaba.com/api/open/queryAgentNoLoginSession  （门店详情 → 免登会话 token）
   ▼
melody.shop.ele.me/auth?ksid=<免登token>&to=/app/shop/<shopId>/stats__center
   │ 免登进入单店经营分析（不需要店铺自己的账号密码）
   ▼
lsycm.alibaba.com/api/naposFlow/queryFlowHistoryData  （进店后单店昨日/30天概览）
   ▼
你的软件解析并展示
```

**关键点**：

1. 服务商端和单店端是**两套 token**。服务商端接口用服务商登录态派生的 `token`；进店后 `queryFlowHistoryData` 用的 `token` 是 `queryAgentNoLoginSession` 返回的**免登会话 token**（即 URL 里的 `ksid`）。
2. 所有接口依赖页面运行时生成的 `secretkey`、`timestamp`、`bx_et`，以及登录态 cookie 和风控上下文。不要裸 HTTP 硬编码这些值，稳定方案是通过真实浏览器页面上下文触发。
3. 服务商身份三件套 `unitId` / `openUserId` / `accountType` 在服务商端每个请求体里都要带。

## 2. 登录与 cookie 复用

### 2.1 登录

服务商账号在 `open.shop.ele.me` 手动登录一次。登录成功后 cookie 落在 `.ele.me` 域，核心 cookie：

| cookie | 作用 |
|--------|------|
| `cna` | 设备指纹 |
| `nrksid` / `yqksid` | 服务商登录态会话 |
| `tfstk` | 风控令牌 |
| `isg` | 风控令牌 |

### 2.2 本地保存（.env）

抓到 cookie 后按 `name=value; ` 拼成一行写入当前目录 `.env`：

```dotenv
ELEME_ISV_COOKIE=cna=...; nrksid=...; yqksid=...; tfstk=...; isg=...
```

同时保存结构化 bundle `eleme-isv-cookie.json`（含每个 cookie 的 domain/path/secure/sameSite/expires），方便一次性用 `Network.setCookies` 恢复完整登录上下文。

> **安全**：`.env`、`eleme-isv-cookie.json` 已加入 `.gitignore`，禁止提交。cookie 值不要写入日志明文。

### 2.3 下次启动注入

用 CDP `Network.setCookies`（传 `eleme-isv-cookie.json` 的 `specs` 数组）注入后再打开控制台，即可免登。cookie 有时效（含风控令牌），过期需重新手动登录刷新。

## 3. 服务商网关通用格式

服务商门店分析接口都是：

```http
POST https://lsycm.alibaba.com/api/oShopAnalysis/open/general/<method>?bx_et=<runtime>
token: <服务商登录态运行时值>
Content-Type: application/json
Referer: https://open.shop.ele.me/
Origin: https://open.shop.ele.me
```

请求体通用身份字段（每个请求都带）：

| 字段 | 说明 |
|------|------|
| `secretkey` | 页面运行时生成，每次不同，不能硬编码 |
| `uniquekey` | 页面会话唯一键，本次会话内固定 |
| `timestamp` | 毫秒时间戳，运行时生成 |
| `unitId` | 服务商单元 ID，本次 `1132809` |
| `openUserId` | 服务商用户 ID，本次 `100100161380818` |
| `accountType` | 账号类型，本次 `member` |
| `source` | 固定 `PC` |

响应通用格式：

```json
{ "code": 0, "message": null, "data": {}, "totalCount": 0, "totalPage": 0 }
```

`code=0` 为成功。

> 抓包中每个业务请求前会先有一条 `GET https://fourier.taobao.com/ts?url=<被包裹的接口URL>` 的埋点信标，可忽略，真实数据在随后的 `POST` 里。

## 4. 接口详解

### 4.1 服务商全店概览（选择时间后重查）

对应动作：进入页面 / 选择"近30日"。

```http
POST https://lsycm.alibaba.com/api/oShopAnalysis/open/general/queryOpenAnalyzeOverview?bx_et=<runtime>
```

请求体（近30日）：

```json
{
  "secretkey": "<runtime>",
  "uniquekey": "<runtime>",
  "timestamp": 1784264864312,
  "shopId": "",
  "source": "PC",
  "unitId": "1132809",
  "openUserId": "100100161380818",
  "accountType": "member",
  "endDate": "20260716",
  "dateType": "recent_30d",
  "beginDate": "20260617",
  "shopIdList": ""
}
```

时间维度取值（`dateType` / 日期）：

| 页面按钮 | dateType | beginDate ~ endDate |
|----------|----------|---------------------|
| 今日实时 | `day` | 今日 ~ 今日（走 `queryOpenRtAnalyzeOverview` 实时接口） |
| 昨日 | `day` | 昨日 ~ 昨日 |
| 近7日 | `recent_7d` | 昨日往前7日 |
| 近30日 | `recent_30d` | 昨日往前30日 |

响应 `data` 是数组，`data[0]` 为全店汇总，核心字段：

| 字段 | 含义 |
|------|------|
| `gross_amt` | 交易额（毛） |
| `income_amt` | 收入额 |
| `valid_order_cnt` | 有效订单数 |
| `exp_uv` | 曝光人数 |
| `clk_exp_rate` | 进店转化率（小数，展示乘 100） |
| `ord_clk_rate` | 下单转化率（小数，展示乘 100） |
| `*_last_week_diff` | 环比上周同期差值 |
| `*_last_period_diff` | 环比前一周期差值 |

> 今日实时用 `queryOpenRtAnalyzeOverview`（同参数结构，`dateType=day`，日期为今日），历史用 `queryOpenAnalyzeOverview`。

### 4.2 门店列表 + 门店搜索（同一接口）

对应动作：门店列表展示 / 搜索框输入 ID 后点放大镜。

> **搜索没有独立接口**：输入框输入时不发请求，点放大镜（或回车）时用 `searchWord` 参数重新调用列表接口。

```http
POST https://lsycm.alibaba.com/api/oShopAnalysis/open/general/pageQueryOpenOfflineDetails?bx_et=<runtime>
```

请求体（搜索门店 ID `1319751638`）：

```json
{
  "pageSize": 10,
  "current": 1,
  "orderBy": "gross_amt",
  "orderType": "desc",
  "unitId": "1132809",
  "openUserId": "100100161380818",
  "accountType": "member",
  "contrastType": "lastWeekDay",
  "key": "recent_30d",
  "beginDate": "20260617",
  "endDate": "20260716",
  "dateType": "recent_30d",
  "dateTypeLabel": "近30日",
  "compareLabel": "比前30日",
  "date": "20260716",
  "searchWord": "1319751638",
  "needDefaultParams": false,
  "shopId": "",
  "source": "PC",
  "secretkey": "<runtime>",
  "uniquekey": "<runtime>",
  "timestamp": "<runtime>"
}
```

| 参数 | 说明 |
|------|------|
| `searchWord` | **门店搜索关键词**，门店 ID 或名称。空串=全部门店 |
| `current` / `pageSize` | 分页，从 1 开始 |
| `orderBy` / `orderType` | 排序字段（默认 `gross_amt` 降序） |
| `dateType` / `beginDate` / `endDate` | 与概览的时间维度联动 |
| `contrastType` | 对比口径，本次 `lastWeekDay` |

响应 `data[]` 每个元素是一家门店（本次命中 1 家，`totalCount=1`），共约 149 个字段，核心：

| 字段 | 含义 |
|------|------|
| `shop_id` | 门店 ID（`1319751638`） |
| `shop_name` | 门店名（小输胜麺屋） |
| `gross_amt` | 交易额（本次 `32337.01`） |
| `valid_order_cnt` | 有效订单数（本次 `681`） |
| `exp_uv` | 曝光人数（本次 `49472`） |
| `clk_uv` | 进店人数（本次 `5065`） |
| `shop_growth_score` | 门店成长分 |
| `package_rating` | 包装评分 |
| `avg_cooking_duration` | 平均出餐时长 |
| `rebuy_rate_7d` | 7 日复购率 |
| `*_last_week_*_diff` / `*_last_period_*_diff` | 各指标环比差值 |
| `*_mall_avg` / `*_mall_avg_rank` | 商圈均值 / 排名 |

> 门店的操作列有两个 `<a>`：**门店详情**、**门店运营**。门店详情走 4.3 免登进店。

### 4.3 门店详情 → 免登进店会话

对应动作：点"门店详情"。**这个接口不返回页面数据，返回一个免登会话 token。**

```http
POST https://lsycm.alibaba.com/api/open/queryAgentNoLoginSession?bx_et=<runtime>
token: <服务商登录态>
```

请求体：

```json
{
  "secretkey": "<runtime>",
  "uniquekey": "<runtime>",
  "timestamp": 1784264958241,
  "shopId": 1319751638,
  "source": "PC",
  "unitId": "1132809",
  "openUserId": "100100161380818",
  "accountType": "member",
  "userAgent": "Mozilla/5.0 (Windows NT 10.0; ...) Chrome/150.0.0.0 Safari/537.36"
}
```

响应：

```json
{
  "code": 0,
  "data": "YJFINJMTAwNTAzOTExMzk2MjAxMTAwMTAwMTYxMzgwODE4UHpuZ1hPMDZUUA__"
}
```

`data` 就是免登会话 token。页面拿到后 `window.open` 跳转：

```text
https://melody.shop.ele.me/auth?ksid=<data>&to=%2Fapp%2Fshop%2F1319751638%2Fstats__center%23app.shop.stats.center
```

即：`melody.shop.ele.me/auth?ksid=<免登token>&to=/app/shop/<shopId>/stats__center#app.shop.stats.center`

| 参数 | 说明 |
|------|------|
| `ksid` | 免登会话 token（= 上面 `data`），进店后作为单店接口的 `token` |
| `to` | 进店后落地路由，`stats__center` = 经营分析中心 |

> **注意**：该跳转用 `window.open` 打开新标签，自动化时可能被浏览器弹窗拦截。稳定做法是拦截 `window.open` 取出 URL，或直接在受控标签 `location.assign` 到该 URL。

### 4.4 进店后：历史数据概览"昨日"

对应动作：进入店铺 → 历史数据概览 → 昨日数据。免登进店落地在 `stats__center`，页面加载单店经营分析全套接口，其中**昨日概览是 `naposFlow/queryFlowHistoryData`**（`dateType=day`）。

```http
POST https://lsycm.alibaba.com/api/naposFlow/queryFlowHistoryData?bx_et=<runtime>
token: <免登会话token，即进店URL的ksid>
Referer: https://melody-stats-next.ele.me/new/
```

请求体（昨日 = 20260716）：

```json
{
  "shopId": 1319751638,
  "secretkey": "<runtime>",
  "timestamp": 1784265090685,
  "key": "day",
  "beginDate": "20260716",
  "endDate": "20260716",
  "dateType": "day",
  "dateTypeLabel": "昨日",
  "compareOption": "LAST_WEEK",
  "compareLabel": "比上周四",
  "bizDate": "20260716",
  "userType": "all"
}
```

| 参数 | 说明 |
|------|------|
| `dateType` | 昨日固定 `day`（近30日则为 `recent_30d`） |
| `dateTypeLabel` | `昨日` |
| `beginDate`/`endDate`/`bizDate` | 都为昨日 `yyyyMMdd` |
| `compareOption` | 昨日对比口径为 `LAST_WEEK`（比上周同一天） |
| `token` | **用免登 ksid**，不是服务商 token |

响应 `data` 是各指标对象（本次昨日实测）：

```json
{
  "code": 0,
  "data": {
    "exp_uv":  {"itemValue":1684, "competitorAvg":611,  "competitorAvg10":2371},
    "clk_exp_rate": {"itemValue":0.077, "competitorAvg":0.072, "competitorAvg10":0.077},
    "ord_uv":  {"itemValue":25,   "competitorAvg":11,   "competitorAvg10":47},
    "exp_pv":  {"itemValue":2160, "competitorAvg":887,  "competitorAvg10":3627},
    "wm_exp_uv": {"itemValue":1684, "competitorAvg":524, "competitorAvg10":2111}
  }
}
```

字段映射（与单店经营分析文档一致）：

| 字段 | 含义 | 单位 |
|------|------|------|
| `exp_uv` | 曝光人数 | 人 |
| `clk_uv` | 进店人数 | 人 |
| `ord_uv` | 下单人数 | 人 |
| `exp_pv` / `clk_pv` / `ord_pv` | 曝光/进店/下单次数 | 次 |
| `clk_exp_rate` | 进店转化率 | 小数，展示乘 100 |
| `ord_clk_rate` | 下单转化率 | 小数，展示乘 100 |
| `wm_*` | 外卖口径同名指标 | |
| `bpt_*` | 团购/到店口径同名指标 | |

每个指标对象字段：

| 字段 | 含义 |
|------|------|
| `itemValue` | 本店昨日值 |
| `resultValue` | 页面展示用结果值 |
| `compareValue` | 对比周期值 |
| `competitorAvg` | 商圈同行均值 |
| `competitorAvg10` | 商圈前10均值 |
| `competitorAvgDiff` / `competitorAvg10Diff` | 与同行/前10 的差值 |

> 进店后同时加载的相关接口（按需取用）：`queryOverviewItems`（商品排行）、`queryOverviewRealTimeData`（今日实时概览）、`serviceTrendAnalyseNew`（服务趋势）、`getPeerInsight`（同行洞察）、`serviceDeliverType`（配送类型）。

## 5. 推荐接入流程

### 5.1 服务商全店看板

1. 从 `.env` 读 cookie，CDP 注入，打开 `open.shop.ele.me/manager/base/store-analysis`。
2. 选时间维度后取 `queryOpenAnalyzeOverview`（历史）或 `queryOpenRtAnalyzeOverview`（今日实时）。
3. 取 `pageQueryOpenOfflineDetails` 得门店列表（可分页遍历）。

### 5.2 搜索指定门店

1. 调 `pageQueryOpenOfflineDetails`，`searchWord` 填门店 ID 或名称。
2. 从 `data[]` 取命中门店的指标。

### 5.3 进店抓单店昨日概览

1. 调 `queryAgentNoLoginSession`（传目标 `shopId`）拿免登 token。
2. `location.assign` 到 `melody.shop.ele.me/auth?ksid=<token>&to=/app/shop/<shopId>/stats__center#app.shop.stats.center`。
3. 进店后调 `naposFlow/queryFlowHistoryData`，`dateType=day`、日期为昨日，`token` 用免登 ksid。
4. 解析 `data` 各指标 `itemValue`。

## 6. 风险与注意事项

| 项目 | 说明 |
|------|------|
| 双 token | 服务商端接口用服务商 token；进店后接口用 `queryAgentNoLoginSession` 返回的免登 ksid |
| 动态参数 | `secretkey` / `timestamp` / `bx_et` / `uniquekey` 均运行时生成，禁止硬编码或复用抓包值 |
| 免登 token 时效 | `queryAgentNoLoginSession` 返回的 token 是短时效会话，进店要尽快用，过期需重新申请 |
| 弹窗拦截 | 门店详情用 `window.open` 跳转，自动化需拦截或改 `location.assign` |
| 服务商身份 | `unitId`/`openUserId`/`accountType` 每个服务商请求都要带，不同服务商账号不同 |
| cookie 安全 | `.env` / `eleme-isv-cookie.json` 已 gitignore，ksid 不落日志明文 |
| 裸 HTTP | 依赖风控上下文，优先浏览器页面上下文触发，不建议后端裸连 |
| 数据来源 | 页面标注"数据来自生意参谋"，指标口径与生意参谋一致 |
